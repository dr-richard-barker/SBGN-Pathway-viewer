
# SBGN Pathway Visualizer

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)
![No API key](https://img.shields.io/badge/API%20key-not%20required-success)
![Runs in browser](https://img.shields.io/badge/runs-in%20your%20browser-informational)

Overlay your RNA-seq (and other omics) data onto **SBGN** and **KEGG** pathway maps, right
in the browser. Upload a gene table, pick a Reactome or KEGG pathway (or bring your own SBGN
file), and the app colours each node by your data on a divergent (fold-change) or sequential
(abundance) scale.

> **100% standalone.** This app no longer depends on Google AI Studio, the Gemini API,
> or any API key. Pathway maps are rendered **deterministically** from real SBGN-ML
> geometry — reproducible and publication-ready — not generated by an LLM.

## What it does

- **Upload omics data** — CSV/TSV with identifiers in the first column (gene symbols,
  Ensembl, Entrez, UniProt; KEGG/ChEBI for compounds).
- **Real pathway maps:**
  - **Reactome** — fetches the genuine, layout-complete SBGN export for any pathway.
  - **KEGG** — fetches the pathway's KGML and renders it directly (gene boxes,
    compound circles, reaction/relation edges), with your data overlaid.
  - **Custom SBGN File** — upload any `.sbgn` map (Reactome, [Newt](https://newteditor.org/),
    VANTED/SBGN-ED, CySBGN, …).
- **Faithful rendering** — SBGN macromolecules, simple chemicals, complexes, compartments,
  processes and the full set of arc types (consumption, production, catalysis, stimulation,
  inhibition, modulation), or KEGG gene/compound/map nodes — all drawn from the file's own
  coordinates.
- **Data overlay + legend** — glyph labels are matched to your identifiers and filled
  from a colour scale; an in-figure legend shows the scale and how many entities mapped.
- **Interactive** — pan, zoom, hover-for-values, search-to-highlight, and **Download SVG**
  (vector, ready for figures).
- **Offline demo** — the **Try offline demo** button renders a bundled example map with
  sample data and **zero network access**, proving the app runs fully self-contained.

## Run locally

**Prerequisites:** [Node.js](https://nodejs.org/) 18+.

```bash
npm install
npm run dev
```

Open the printed URL (default <http://localhost:5173>). No `.env`, no API key.

To build a static bundle and preview it:

```bash
npm run build      # outputs to ./dist
npm run preview
```

## Deploy it so anyone can use it

### Option A — GitHub Pages (automated, website + app)

The workflow [`.github/workflows/deploy.yml`](.github/workflows/deploy.yml) builds **both** the
Jekyll site and the Vite app and publishes them together. Once:

1. In your repo: **Settings → Pages → Build and deployment → Source = GitHub Actions**.
2. Push to `main`. The action builds the website (root) and the app (`/app/`) and deploys.
3. Live at `https://<user>.github.io/<repo>/` with the app at `…/<repo>/app/`.

### Option B — app only (manual / other hosts)

The app build is a plain static site with relative asset paths:

```bash
npm run build        # outputs to ./dist
npm run preview      # local check
npm run deploy       # optional: publish ./dist to the gh-pages branch (gh-pages dev dep)
```

Or on Netlify / Vercel: build command `npm run build`, publish directory `dist`.

## Data format

The first column is the identifier; matching to glyphs is by label (case-insensitive,
whole-label first, then word tokens).

**Log2 fold change** (divergent blue↔red scale) — a column matching `log2FoldChange`/`lfc` is used:

```csv
SYMBOL,baseMean,log2FoldChange,padj
CDC20,150.5,2.75,0.0001
AURKB,89.2,-1.98,0.0023
```

**Normalized counts / abundance** (sequential scale) — numeric columns are averaged:

```csv
SYMBOL,Sample1,Sample2,Sample3
CDC20,512,480,550
AURKB,10,25,15
```

Compounds work the same way with KEGG/ChEBI IDs in the first column.

## Notes on other databases

**Reactome** (SBGN) and **KEGG** (KGML) render directly. MetaCyc/BioCyc, SMPDB, PANTHER and
MetaCrop are still listed for browsing species and pathways, but they do **not** publish
SBGN-ML or KGML, so to map data onto one of those pathways, export it to SBGN (e.g. via Newt
or CySBGN) and load it through **Custom SBGN File**.

KEGG matching: your gene identifiers are matched against the KGML node's symbols
(`graphics name`, e.g. `ENO1`) **and** its database ids with the organism prefix stripped
(`ath:AT1G12345` → `AT1G12345`, `hsa:7157` → `7157`), so gene symbols, locus ids and Entrez
ids all work. Compounds match KEGG `Cxxxxx` ids. KEGG REST has no CORS headers, so KGML is
fetched through a free public proxy (`api.allorigins.win`).

## How it works

```
upload CSV ─▶ parseDataToMap ─────────────┐
                                          ├─▶ renderSbgnToSvg / renderKgmlToSvg
choose source ─▶ getPathwaySource ────────┘        │  ▲
  (Reactome SBGN / KEGG KGML / custom / demo)       │  └─ shared overlay (match + colour + legend)
                                                    └─▶ inline SVG (legend, glyph-*/data-omics-* ids)
```

Key modules in [`services/`](services/): `sbgnSource.ts` (fetch/resolve SBGN-ML or KGML),
`sbgnRenderer.ts` (SBGN-ML → SVG), `kgmlRenderer.ts` (KEGG KGML → SVG), `overlay.ts`
(shared value-matching, colour scales, legend), `colorScales.ts`, `pathwayRenderer.ts`
(orchestration/dispatch). The old `geminiService.ts` has been removed.

## FAIR & accessibility

This project is built to be **FAIR** (Findable, Accessible, Interoperable, Reusable) and
WCAG-minded:

- **Findable** — rich page metadata + Open Graph tags, [`CITATION.cff`](CITATION.cff) and
  [`codemeta.json`](codemeta.json) machine-readable records, descriptive README and keywords.
- **Accessible** — free, open, no login or API key; runs entirely client-side and deploys as
  a static site anyone can open. Keyboard-operable controls, ARIA labels/roles, a skip link,
  live status announcements, visible focus, `prefers-reduced-motion` support, and
  contrast-aware glyph labels.
- **Interoperable** — consumes community-standard **SBGN-ML** and **KEGG KGML**; exports
  standard **SVG**. Identifiers are matched across symbols, locus ids and Entrez/KEGG ids.
- **Reusable** — permissive **MIT** license, versioned, documented, dependency-light, and
  the output SVG is self-contained (KEGG images are embedded as data URIs).

## Releasing & minting a Zenodo DOI

A [`.zenodo.json`](.zenodo.json) describes the deposit metadata. To get a persistent,
citable archive (recommended for FAIR):

1. Add your **ORCID** and affiliation in [`.zenodo.json`](.zenodo.json) (`creators[].orcid`,
   `creators[].affiliation`) and uncomment the ORCID line in [`CITATION.cff`](CITATION.cff).
2. Sign in at [zenodo.org](https://zenodo.org) with GitHub, open **Account → GitHub**, and
   flip the switch **on** for `dr-richard-barker/SBGN-Pathway-viewer`.
3. On GitHub, **Releases → Draft a new release**, tag `v1.0.0`, publish. Zenodo catches the
   release webhook and mints a DOI automatically.
4. Copy the **concept DOI** (the "all versions" one) into the badge below, into
   `CITATION.cff` (`doi:` / `identifiers:`), and into `.zenodo.json` if you wish:

   ```markdown
   [![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.XXXXXXX.svg)](https://doi.org/10.5281/zenodo.XXXXXXX)
   ```

Each later GitHub release then gets its own version DOI under the same concept DOI.
